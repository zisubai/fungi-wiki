# 菌百科 MVP 交付验收

## 完成范围

当前版本完成了可交付 MVP：公开菌种百科、结构化功能菌数据库、多条件与容错搜索、可解释菌种/组合推荐，以及带角色权限的运营管理端。运营流程覆盖批量导入、数据质量、功能与培养条件维护、文献证据、应用案例、数据版本快照、审核发布、推荐反馈和共培养实验回流。

合成生物设计平台中的代谢通路、基因模块、项目空间和多组学分析属于后续产品阶段，不属于本次 MVP 完成范围。

## 自动验收

```bash
npm run release:check
```

该命令执行前端覆盖率测试与生产构建、Go 测试和静态分析、API 编译、Docker Compose 配置校验；本地服务已运行时还会执行 HTTP 冒烟检查。

## 人工验收

1. 在用户端按功能、温度、pH、安全等级和来源环境联合搜索。
2. 打开菌种详情，检查功能、培养条件、别名和文献证据。
3. 对 2–3 个已发布菌种执行横向对比。
4. 从功能菌专题库进入一个功能方向，确认已发布数量和筛选结果一致。
5. 在运营端新增应用案例，确认用户详情页展示行业、场景、方案、结果、成熟度和来源。
6. 修改菌种或关联数据，确认“应用与版本”自动新增版本记录。
7. 提交自然语言菌种推荐和双功能组合推荐，检查理由、证据、风险与实验状态。
8. 登录运营端，新建草稿并维护关联数据，提交审核后由专家或管理员发布。
9. 导入 CSV/Excel，确认逐行校验与批次记录。
10. 在推荐质量页录入候选级共培养实验，重新推荐并确认实验结果影响排序。
11. 配置 embedding 服务并重建向量索引，使用自然语言需求确认响应中 `semanticEnabled=true` 且存在“语义相似”匹配原因。
12. 登录用户端收藏菌种、执行搜索并重新登录，确认收藏和搜索历史可恢复。
13. 在运营端新增同义词与召回规则，确认查询扩展和排序变化。
14. 运行 `REQUIRE_SEMANTIC=1 npm run eval:search`，确认标准评测集 Recall@5 达到门禁。

## 发布前要求

- 替换 `JWT_SECRET`、管理员密码和数据库密码。
- 配置正式域名对应的 `CORS_ALLOWED_ORIGINS`。
- 替换或删除演示文献及 `demo-seed` 数据。
- 为 PostgreSQL 配置持久化备份、恢复演练和访问控制。
- 在真实科研数据上线前完成人工证据审核和生物安全审查。
