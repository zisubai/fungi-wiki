# 微生物百科数据库表结构设计

## 1. 设计目标

数据库围绕“菌种主数据 + 功能标签 + 文献证据 + 实验条件 + 应用案例 + 审核流 + 搜索推荐日志”设计，优先满足 MVP 和功能菌数据库阶段，后续可扩展到 AI 推荐和合成生物设计辅助平台。

## 2. 核心实体

- `species`：菌种主表。
- `species_aliases`：菌种别名、旧名、中文别名。
- `taxonomies`：分类体系。
- `function_tags`：功能标签。
- `species_functions`：菌种与功能关联。
- `culture_conditions`：培养和环境条件。
- `literatures`：文献、专利和外部证据来源。
- `evidences`：证据表，连接菌种、功能和文献。
- `application_cases`：应用案例。
- `audit_records`：审核记录。
- `search_logs`：搜索日志。
- `recommendation_records`：AI 推荐记录。
- `user_feedback`：用户反馈。

## 3. 表关系

```text
species 1 - N species_aliases
species N - 1 taxonomies
species N - N function_tags，通过 species_functions 关联
species 1 - N culture_conditions
species 1 - N evidences
function_tags 1 - N evidences
literatures 1 - N evidences
species 1 - N application_cases
species 1 - N audit_records
species 1 - N recommendation_records
```

## 4. 枚举建议

菌种状态 `species.status`：

- `draft`：草稿。
- `pending_review`：待审核。
- `published`：已发布。
- `archived`：已归档。

审核状态 `audit_records.status`：

- `pending`：待审核。
- `approved`：已通过。
- `rejected`：已拒绝。

证据等级 `evidences.evidence_level`：

- `low`：低。
- `medium`：中。
- `high`：高。
- `expert_verified`：专家确认。

## 6. 后续扩展

智能搜索阶段：

- 给 `species.summary`、`species_functions.description`、`literatures.abstract` 增加全文索引。
- 增加 `species_embeddings` 表保存向量。
- 使用 `pgvector` 或外部 Milvus 存储向量索引。

AI 推荐阶段：

- 扩展 `recommendation_records`，保存召回候选、评分明细和证据引用。
- 增加 `recommendation_feedback` 或复用 `user_feedback`。
- 增加高风险规则表。

合成生物设计阶段：

- 增加 `chassis_organisms` 底盘菌表。
- 增加 `pathways` 代谢通路表。
- 增加 `genes` 基因表。
- 增加 `genetic_parts` 基因元件表。
- 增加 `design_projects` 项目空间表。
