# 本地开发启动说明

## 1. 服务地址

本地开发默认使用以下端口：

```text
API 后端：      http://localhost:8080
用户前端：     http://localhost:5173
运营管理端：   http://localhost:5174
PostgreSQL：   localhost:55432
```

## 2. 环境变量

根目录后端环境变量示例：

```text
.env.example
```

用户前端环境变量示例：

```text
apps/web/.env.example
```

运营端环境变量示例：

```text
apps/admin/.env.example
```

前端和运营端都使用：

```text
VITE_API_BASE_URL=http://localhost:8080
```

如果后端部署在其他地址，可以复制 `.env.example` 为 `.env.local` 后修改：

```bash
cp apps/web/.env.example apps/web/.env.local
cp apps/admin/.env.example apps/admin/.env.local
```

## 3. 一键开发启动

项目提供了一键开发脚本：

```bash
./scripts/dev.sh
```

脚本会尝试执行：

- 检查 `npm` 和 `go`。
- 如果 Docker daemon 可用，则启动 PostgreSQL。
- 启动 Go API。
- 启动用户前端。
- 启动运营管理端。

启动后访问：

```text
用户前端：   http://localhost:5173
运营管理端： http://localhost:5174
API 健康检查：http://localhost:8080/healthz
```

停止服务：

```text
Ctrl+C
```

## 4. 手动启动

如果不使用一键脚本，可以分三个终端启动。

### 4.1 启动数据库

```bash
./scripts/db-up.sh
```

如果提示 Docker 未启动，需要先打开 Docker Desktop。

### 4.2 启动 API

```bash
npm run dev:api
```

默认数据库连接：

```text
postgres://fungi:fungi@localhost:55432/fungi_wiki?sslmode=disable
```

### 4.3 启动用户前端

```bash
npm run dev:web
```

### 4.4 启动运营管理端

```bash
npm run dev:admin
```

## 6. 常见问题

### 6.1 Docker daemon is not running

说明 Docker Desktop 没有启动。处理方式：

- 打开 Docker Desktop。
- 等待 Docker 状态变为 running。
- 重新执行 `./scripts/db-up.sh` 或 `./scripts/dev.sh`。

### 6.2 role "fungi" does not exist

通常是连接到了本机已有的 PostgreSQL 5432，而不是项目容器。项目默认使用 55432 端口，请确认：

```text
DATABASE_URL=postgres://fungi:fungi@localhost:55432/fungi_wiki?sslmode=disable
```

### 6.3 前端请求后端失败

确认前端环境变量：

```text
VITE_API_BASE_URL=http://localhost:8080
```

确认 API 正常：

```bash
curl http://localhost:8080/healthz
```

### 6.4 用户前端列表为空

用户前端只显示 `published` 状态的菌种。可以在运营端新增菌种，并把状态设置为 `已发布`。
